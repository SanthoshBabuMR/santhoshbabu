Data type

  • All values in javaScript are either primitives or objects.
  • Primitive includes string, number, boolean, undefined, null and symbol
  • Any non-primtive value expect null is an object
  • Please not a value is immutable while a variable is not. A variable holds a reference to their (primitive) value.
  • more on data type

typeof

  • All values in javaScript are either primitives or objects. typeof operator returns the type of value
  • For primitive value, returns the value type
  • For any non-primtive value expect null, typeof returns value type as object;
  • Example: 
    typeof "foo";            // "string"
typeof true;             // "boolean"
typeof -1262;            // "number"
typeof function fn(){};  // "function"
typeof undefined;        // "undefined"
    typeof {};      // "object"
typeof [];      // "object"
typeof null;    // "object"

Evalution to true

  • All values are truthful, except the following
    1. ""
    2. 0
    3. false
    4. null
    5. undefined
    6. NaN

undefined

  • represents "no value"
    1. Uninitialized variables are undefined 
      var foo;
console.log(foo); // undefined
    2. Missing params in functions are undefined 
      function fn(x) {
  return x;
}
fn(); // undefined
    3. Reading a non existent property in an object returns undefined 
      var obj = {};
console.log( obj.name ); // undefined

Dynamic typing

  • A variable in javascript can be assigned with different data types. 
    var foo = "bar";    // foo holds string data type
    foo = false;    // foo holds boolean data type
    foo = [1,2,3];  // foo holds object data type

ECMAScript

  • ECMAScript is a scripting-language specification standardized by ECMA International
  • JavaScript is an implementation of ECMAScript
  • ECMA stands for "European Computer Manufacturers Association"
